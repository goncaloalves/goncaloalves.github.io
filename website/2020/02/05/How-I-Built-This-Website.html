How I built this website

Feb 5, 2020 • Gonçalo • Website

My idea this website, my personal website, was to build a good looking and secure website for no cost at all. And for that I used the following stack:

  • Dreamhost - Domain name
  • Github - Website Hosting
  • Jekyll - Static site generator

Dreamhost - Domain name

This one was a really simple choice: I had already bought a few domain names as well as website hosting from Dreamhost and really liked their costumer service. They make it simple to work with a domain name without web hosting from them.

This particular domain name was not my first choice, but I still like it. My first choice, unfortunately was already taken.

I believe that there are no cons to this solution I’ve chosen for domain name since it´s pretty much similar to every other option out there. I guess the value in the solution comes from the price and the user experience that is provided via Dreamhost’s website and customer support.

Github - Website Hosting

I really didn’t want to deal with the traditional web hosting methods of FTP or SSH, not because I’m not comfortable with them, but because I find them cumbersome. So when I found out that GitHub had their own web hosting solution, GitHub Pages, and that they allowed for custom domains, that was an easy option for me since all I had to do to publish the website was do a git push.

I also had a concern about security, I didn’t want to be worried about someone hacking or defacing this website, so, in this case, I let GitHub Pages to manage the security for me (althouth there are still some aspects of security that I have to tend to).

The cons of this solution is that this is not for everyone. Although it’s pretty simple to use, I believe it requires a tech savvy mind. I guess the that the fact that this solution is provided by GitHub, a platform for developers, is a pretty solid indicator that is not for everyone.

Jekyll - Static Site Generator

As I’m writing this page and look to the GitHub Pages website, I see that they have a section dedicated to using Jekyll to host a website on GitHub Pages.

As I’ve stated in the previous point, security was a concern to me. As Jekyll is a static site generator, there is no database to hack and so SQL injection and Cross site-scripting (which are some of the most common vulnerabilities) have been removed from the attack surface of this website. That is not to say that this website is un-hackable, but to simply state that it is a fair bit more secure.

Another advantages of static site generators are performance and fast page load. Since Jekyll renders pages to static HTML files there are little server resources to use and, because of this, I expect a good user experience.

As in the previous point, Jekyll is fairly complicated for those without a programmer experience. Jekyll requires some code to get it to work, and it doesn’t have the bells and whistles of Wordpress, with all the plugins, community and Gutenberg, the WYSIWYG editor that makes editing content a pleasure.
